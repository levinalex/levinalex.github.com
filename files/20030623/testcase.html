    <div class="outer">
    <div class="middle">
    <div class="inner">text</div>
    </div>
    </div>

  2. replaced div.middle with a.middle

    The Validator complains about div.inner. This is because inline elements may contain only data and other inline elements in HTML4.1

    Two Screenshots that show what Opera7/Win does and what Mozilla1.3b/Win does

    Note that the CSS-Rule .middle .inner{text-decoration:underline;} stopped working.

    <div class="outer"> <a class="middle" style="display:block;">
    <div class="inner">text</div>
    </a>     </div>

  3. Nearly the same as #12, I threw in an <ins>-Element. Still illegal HTML 4 but the Validator cannot detect it.

    <div class="outer"> <a class="middle" style="display:block;"> <ins>
    <div class="inner">text</div>
    </ins>     </a>     </div>

  4. I replaced the innermost div with a span. Works now.

    <div class="outer"> <a class="middle"> <span class="inner" style="display:block;">text</span> </a> </div>

  5. This is as illegal as it gets. Opera and Mozilla agree to render this as three seperate Blocks

    <a class="outer"> <a class="middle"> <a class="inner">text</a> </a> </a>
